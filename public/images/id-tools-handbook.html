Before You Open Any Tool

Instructional Design Fundamentals

Understanding the theory and process behind effective ID is what separates well-designed learning from a decorated slide deck.

💡 What is Instructional Design?

Instructional design (ID) is the systematic process of creating educational and training experiences so that knowledge and skills are acquired more efficiently and effectively. IDs draw on learning theory, cognitive psychology, and communication design to build materials that actually change behavior — not just deliver information.

The ADDIE Model

ADDIE is the most widely used ID framework. Every tool in this handbook fits into one of these five phases.

A

Analyze

Identify the learning gap, audience characteristics, existing knowledge, constraints, and goals. Ask: Who are the learners? What do they need to be able to do that they can't do now? What's causing the gap?

D

Design

Write learning objectives, map the content structure, plan assessments, and decide on the instructional strategy. Create your storyboard here — in Google Docs and Figma before touching an authoring tool.

D

Develop

Build the actual course using Articulate Storyline, Rise, or Captivate. Develop slides, add interactions, record narration, create assessments, and produce all supporting materials in Canva or Figma.

I

Implement

Deploy the course through your LMS (Moodle, Canvas, etc.). Set up enrollments, configure SCORM tracking, communicate with learners, and ensure technical support is ready for the launch.

E

Evaluate

Measure effectiveness using Kirkpatrick's Four Levels: Reaction (did learners enjoy it?), Learning (did they gain knowledge?), Behavior (did they apply it?), and Results (did it solve the business problem?). Use LMS reports and Google Forms surveys to collect data.

Writing Strong Learning Objectives
✅
Use Bloom's Taxonomy verbs

Good objectives use measurable action verbs from Bloom's Taxonomy: Remember (recall, list), Understand (explain, describe), Apply (use, execute), Analyze (compare, differentiate), Evaluate (judge, justify), Create (design, develop). Instead of "learners will understand safety procedures," write: "Learners will be able to identify and apply the three-step safety protocol when operating equipment."

Storyboarding Before You Build
1
Plan

Create your course outline

List all modules, lessons, and topic points in a simple hierarchy. A typical e-learning course has 3–5 modules, each with 2–5 lessons. Use a Google Doc with Heading 1 for modules, Heading 2 for lessons, and bullet points for slide-level content.

2
Plan

Choose your storyboard format

For most projects, use a two-column storyboard (Visual Description | Audio/Text Script). For courses with branching scenarios, add a third column for logic/triggers. For complex multi-path experiences, use a flowchart storyboard in Figma showing decision branches.

3
Plan

Write all content before designing

Draft the full script and on-screen text in your storyboard before opening any design tool. Content drives design. Share the storyboard with your SME (Subject Matter Expert) for review using Google Docs' Comment feature — resolve all feedback before proceeding to development.

4
Plan

Plan your assessments from the start

Write your quiz questions alongside the content — never as an afterthought. Map each question back to a specific learning objective. Plan for varied types: multiple choice, drag-and-drop, scenario-based decision-making, and reflection prompts. Great assessments measure actual performance, not just recall.

⚠️
Never skip the Analyze phase

The biggest mistake novice IDs make is jumping straight into tool use without properly understanding the problem. A course nobody needed is a course nobody completes. Spend time interviewing stakeholders and sampling real learners first.

Authoring Tool

Articulate 360

The gold standard in e-learning authoring. Storyline 360 for powerful custom courses; Rise 360 for rapid, responsive content.

📖

Articulate Storyline 360

PowerPoint-like interface for building fully custom, interactive e-learning with timelines, triggers, layers, and states.

1
Setup

Download and install Articulate 360

Go to articulate.com → start a 60-day free trial or sign in with a licensed account. Download the Articulate 360 desktop app, which is the launcher for all Articulate tools. From there, install Storyline 360.

2
Setup

Create a new project

Open Storyline 360 → click New Project → under Story Size, choose 16:9 Widescreen (1280×720px) — the most common and LMS-friendly format. Give it a name and save immediately to a local drive (not a network or cloud drive, as Storyline can behave unpredictably with synced folders).

3
Interface

Learn the four key panels

Slide Panel (left): all your slides, organized into Scenes. Stage (center): your visual canvas. Timeline (bottom): controls when each object appears, disappears, and animates. Triggers Panel (right): the brain of your course — every interaction is a trigger. Spend 20 minutes clicking around before building anything.

4
Build

Organize slides into Scenes

In the Slide Panel, right-click → New Scene to create a container for each module (e.g., Scene 1 = Introduction, Scene 2 = Module 1). Then right-click inside a scene to Add New Slide. This keeps large courses manageable. You can import PowerPoint slides via Insert → Import → PowerPoint.

5
Build

Add content to slides

Use the Insert tab to add text boxes, shapes, images, video, audio, and characters. From the Content Library 360, access thousands of built-in illustrated characters, backgrounds, and icons. Position objects by clicking and dragging. Hold Shift while dragging to constrain movement to horizontal or vertical.

6
Interactivity

Add triggers and states

Triggers are the heart of Storyline interactivity. In the Triggers panel → Create New Trigger. The three parts of every trigger are: Action (what happens), Object (what it happens to), and When/Condition (what sets it off). Common example: "Jump to Slide 1.2 when the user clicks Button 1." States allow you to change how any object looks — Normal, Hover, Selected, Visited, Disabled.

7
Interactivity

Use layers for pop-ups and feedback

Layers sit on top of your base slide and are shown or hidden by triggers. Perfect for feedback panels, modal windows, and tab interactions. Add via Insert → New Layer. Each layer has its own independent timeline and can have its own objects. A common pattern: base slide has buttons; each button shows a different layer with detail content.

8
Assess

Build your quiz

Go to Insert → New Slide → Graded Question. Choose from Multiple Choice, True/False, Matching, Drag & Drop, Sequence, Short Answer, and more. After adding questions, insert a Result Slide (Insert → New Slide → Result Slide). In the Result Slide properties, set your passing score and configure what happens on pass or fail (e.g., "Continue to next section" vs. "Jump to Review Module").

9
Publish

Preview and publish your course

Press F12 to preview the full project in Storyline's built-in player. For a browser preview, click Preview → Preview Project. To publish: Home → Publish. For LMS delivery, choose LMS and select SCORM 1.2 (most universally supported). Click Publish → you'll get a ZIP file. Upload that ZIP as-is to your LMS.

Articulate Rise 360

Browser-based rapid authoring for beautiful, responsive courses — no software install needed.

1
Setup

Access Rise — no download required

Rise is entirely browser-based. Log into articulate.com → My Articulate → click Rise 360. Create a new course by clicking + New Course. Enter a title and description to help your team identify it.

2
Build

Structure your course with Sections and Lessons

Rise organizes content into Sections (modules) and Lessons (individual pages). Add a Section for each main topic, then nest Lessons inside. Drag and drop to reorder. You can also add a Quiz lesson type for graded assessment.

3
Build

Build lessons with content blocks

Inside each lesson, click the + icon to add blocks. Rise offers over 40 block types: Text, Image, Video, Button, Quote, Process, Accordion, Tabs, Sorting Activity, Flashcards, Timeline, Scenario, and more. Each block is already mobile-responsive and beautifully styled — just add your content.

4
Design

Customize your brand theme

Click Theme in the top toolbar. Set your brand accent color, fonts, and cover style. Upload your logo. Rise's themes are elegant by default — small color tweaks go a long way. For more control, use Custom Fonts to upload your brand typefaces.

5
Assess

Add quizzes and knowledge checks

To embed a quick check inside a lesson, add a Knowledge Check block. For a formal graded quiz, add a separate Quiz lesson — this tracks completion differently. Set question types, feedback, and the passing score in the Quiz settings panel.

6
Publish

Preview and publish

Click Preview at the top to see your course in a simulated browser, tablet, and mobile view simultaneously. When ready, click Publish → LMS and download your SCORM 1.2 ZIP. Alternatively, Publish for Web generates an instant shareable link — perfect for stakeholder reviews without LMS setup.

Figma for Instructional Designers

Figma is where you design and prototype before building. Used for wireframing, storyboarding, visual asset creation, and stakeholder previews.

ℹ️
Why Figma before your authoring tool?

Designing your slide layouts and interactions in Figma first lets you get fast stakeholder feedback before spending hours building in Storyline. A 30-minute Figma prototype can prevent 10 hours of rework. You also export background images and assets directly from Figma into your authoring tool.

1
Setup

Create a free Figma account

Go to figma.com → sign up for free. The Starter plan (free) allows 3 Figma files with unlimited collaborators — plenty to start with. Download the optional desktop app for better performance, or use it entirely in your browser.

2
Interface

Learn the Figma workspace

Left panel: Layers (object hierarchy) and Pages (like tabs in a file). Canvas: infinite work area — scroll to navigate, Ctrl+scroll to zoom. Right panel: Design properties (size, color, effects) and Prototype tab (interactions). Toolbar: Move V, Frame F, Rectangle R, Text T, Pen P.

3
Build

Create frames sized for e-learning

Press F → in the right panel under Frame presets, choose 1280 × 720 (Storyline's standard canvas) or type custom dimensions. Double-click the frame name to rename it (e.g., "Slide 01 – Title"). Create one frame per slide in your storyboard — you'll end up with a full visual storyboard in one Figma file.

4
Build

Design slide layouts with grids

Select a frame → right panel → Layout Grid → + → choose Grid: 8px for alignment snapping. Design backgrounds with rectangles, headings with text boxes, and icons from Figma's built-in Resources panel (press the grid icon or use the plugin Iconify for thousands of free icons).

5
Build

Build reusable components

Select any element → right-click → Create Component (Ctrl+Alt+K). Common components to create: navigation bar, slide header, info button, character frame, and next/back buttons. Instances of components (drag from the Assets panel) inherit changes from the master — update once, update everywhere.

6
Design

Create color and text styles

Select a filled shape → right panel → Fill → click the 4-dot icon → +Create Style. Name it by category: "Brand/Primary", "Brand/Background", "State/Hover". Do the same for typography. Styles ensure visual consistency across all slides and make global rebrand changes trivial.

7
Prototype

Create a clickable prototype

Switch to Prototype mode (top-right tab, or Shift+E). Click any element → drag the blue node to a target frame to create a link. Set the trigger (On Click) and transition (Dissolve, Slide In Left). Press ▶ Present to launch the interactive prototype. Share the link with stakeholders — they click through and leave comments without needing a Figma account.

8
Export

Export assets for your authoring tool

Select any frame or element → right panel → Export → +. Choose format: PNG (background images, illustrations), SVG (icons, shapes that need to scale), or PDF (full slides). To bulk export all frames, select all → Export. These assets drop directly into Storyline's media library or Captivate's Library.

9
Collaborate

Share for review and feedback

Click Share (top-right) → set to "can view" for SMEs and stakeholders. They open the link, switch to Comment mode (keyboard shortcut C), and click anywhere on the designs to leave feedback. You see all comments in the right panel and can resolve them as you work — far faster than email rounds.

Canva for Educational Materials

Create polished supporting materials — infographics, job aids, worksheets, and more — with drag-and-drop simplicity. No design background required.

1
Setup

Create your Canva account

Go to canva.com and sign up for free. If you're working in an educational institution, apply for Canva for Education — it gives you free Pro access with the full asset library. The free plan works well for most ID needs; Pro adds Brand Kit, background removal, and premium templates.

2
Setup

Set up your Brand Kit

Go to Brand Hub in the left sidebar → upload your organization's logo, add your brand colors (paste hex codes), and choose your brand fonts. With Pro, apply the Brand Kit to any design in one click. Even on the free plan, saving a manual color palette saves significant time.

3
Build

Choose the right canvas size

Click Create a design → search for your format. Key ID formats: Presentation (16:9) for slide decks, Infographic (800×2000) for visual summaries, A4 Document for worksheets and handouts, Whiteboard for collaborative planning. You can also type custom dimensions for LMS banners or email headers.

4
Build

Design infographics for learning

Infographics make excellent module summaries and concept explainers. Start from an infographic template → clear out the placeholder text → add your own. Use the Elements panel for charts, icons, lines, and shapes. Stick to 3–5 key points per infographic. Maintain one clear reading flow from top to bottom.

5
Build

Create job aids and quick reference cards

Job aids are among the highest-ROI deliverables in ID — learners reference them on the job long after the course ends. Use a structured layout: numbered steps or a clear table. Keep it to one page (A4 portrait or landscape). Use icons from the Elements panel to replace long text blocks with visual cues. Aim for print-and-post readability.

6
Export

Download and share your materials

Click Share → Download. For print handouts: PDF Print (CMYK, highest quality). For digital distribution: PDF Standard or PNG. To present live, use Share → Present directly from Canva. For team access, click Share → Invite People and give editing access to collaborators on your team.

💡
What to make in Canva as an ID

LMS module banners and cover images, participant workbooks, certificate templates, facilitator guides, onboarding welcome cards, pre/post-test instruction sheets, social media posts announcing your course launch, and visual learning objectives slide decks.

05
Authoring Tool

Adobe Captivate

Captivate shines for software simulations, responsive fluid box design, and VR/360° learning environments. Steeper curve, but uniquely powerful for the right projects.

ℹ️
Choose Captivate when you need to:

Record on-screen software simulations (e.g., training people on a new CRM), build truly responsive content that reflows automatically across device sizes using Fluid Boxes, or create immersive VR/360° learning environments. For standard slide-based e-learning, Articulate Storyline is usually faster.

1
Setup

Install Adobe Captivate

Go to adobe.com/products/captivate. Download a 30-day free trial or purchase a subscription ($33/mo). Install and launch. On the Start screen, click New Project. Captivate uses your local drive for project files — save early and often.

2
Setup

Choose your project type

Captivate offers: Blank Project (manual slide-based), Software Simulation (records your screen with click detection), Responsive Project (fluid box layout for multi-device), Video Demo (screen recording as MP4), or VR Project (360° environment). For your first project, start with a Blank Project at 1280×720.

3
Build

Record a software simulation

Go to File → Record New Software Simulation. Open the software you want to demonstrate, define your capture area (snap to the application window), choose Demo mode (auto-generated callouts) or Training mode (click boxes where learners must click). Click Record, perform the steps, then stop. Captivate generates a slide for each action automatically.

4
Build

Add interactive elements

Use the Interactions panel (right sidebar) to insert Buttons, Click Boxes (invisible hotspots), Text Entry Boxes, and Rollovers. Objects are controlled by the Timeline at the bottom — drag object bars to control when they appear. Add Success Caption and Failure Caption feedback to any interactive element via its Properties panel.

5
Assess

Build a quiz

Go to Quiz → Question Slide. Captivate supports Multiple Choice, Fill in the Blank, True/False, Matching, Sequence, Short Answer, Hotspot, and Likert Scale. In the Quiz Properties panel (right side), set pass/fail scores, number of attempts, and post-quiz branching. Captivate can send quiz results to your LMS via SCORM.

6
Publish

Publish for your LMS

Go to File → Publish for Devices → select LMS. Choose SCORM 1.2 for maximum compatibility, or SCORM 2004 if your LMS specifically requires it. Configure completion criteria (slide viewed, quiz score, etc.), then click Publish. Upload the generated ZIP to your LMS — the same process as Storyline.

Collaboration & Content Development

Google Workspace for ID

Docs, Slides, Forms, and Drive are the backbone of your ID workflow — free, cloud-based, and built for the kind of collaboration that ID demands.

📄 Google Docs — Storyboarding & SME Collaboration ▼
1

Set up your storyboard template

Use a two-column table: Column 1 = Visual Description, Column 2 = Audio/On-Screen Text. Use Heading 1 for module names, Heading 2 for lesson names, and Heading 3 for individual slide titles. This auto-generates a clickable Table of Contents (Insert → Table of Contents).

2

Collaborate with SMEs using Comments and Suggestions

Share with your SME as "Commenter." They highlight text + press Ctrl+Alt+M to leave inline comments. Switch to Suggesting mode (pencil icon → "Suggesting") so you can see exactly what they changed before accepting. Resolve comments as you address them — this creates a clean revision trail.

3

Use Version History to track drafts

Go to File → Version History → See Version History. Name important versions (e.g., "SME Review v1," "Approved Final"). You can restore any previous version at any time — no more emailing files named "storyboard_FINAL_v3_ACTUALFINAL."

📊 Google Slides — Rapid Presentation Development ▼
1

Design with a custom theme/master

Go to Slide → Edit Theme to set brand fonts, colors, and layouts that apply across all slides. This is your slide master — any change here ripples through the entire deck. Far more reliable than manually applying formatting to each slide.

2

Import into Storyline as a starting point

Download your presentation as PPTX (File → Download → Microsoft PowerPoint). Then in Storyline: Insert → Import → PowerPoint. Storyline brings in all slides while preserving fonts, images, and layouts. This workflow lets you draft content quickly in Slides and polish the interactivity in Storyline.

📋 Google Forms — Needs Analysis & Evaluations ▼
1

Build a needs analysis survey

Go to forms.google.com → Blank form. Add questions: multiple choice for closed questions, short answer for open-ended, and Linear Scale (1–5) for rating satisfaction. Sections let you create different question paths based on responses (Settings → Sections). Share the link or embed it in your LMS.

2

Use Quiz Mode for graded assessments

Go to Settings → Make this a quiz. Set point values for each question and add answer feedback. Google Forms auto-grades multiple choice and checkbox questions. Connect to a Google Sheet (Responses → Spreadsheet icon) to analyze all results in one place — filter by score, date, or respondent.

🗂️ Google Drive — Project Organization ▼

Organize every ID project the same way — build a folder template in Drive and copy it for each new project:

Uploading SCORM to Your LMS — Step by Step
1
Know This

Understand SCORM basics

SCORM (Shareable Content Object Reference Model) is the packaging standard that lets any LMS "understand" an e-learning course — tracking completion, scores, and time. The two common versions are SCORM 1.2 (widest compatibility — use this as your default) and SCORM 2004 (more data tracking, but fewer LMS support it fully). When in doubt, SCORM 1.2 is safe.

2
Publish

Export the SCORM package from your authoring tool

In Storyline: Home → Publish → LMS → SCORM 1.2 → Publish. In Rise: Publish → LMS → SCORM 1.2 → Download. In Captivate: File → Publish for Devices → LMS → SCORM 1.2 → Publish. In all cases, you receive a ZIP file. Do not unzip it — your LMS needs the ZIP package as-is.

3
Upload

Upload to your LMS

In Moodle: Go to your course → Turn Editing On → Add an Activity → SCORM Package → drag in your ZIP. In Canvas: Modules → + → External Tool → upload, or use Files → upload ZIP and create a content page. In TalentLMS: Courses → Add Course → Content → Add Unit → SCORM. Each LMS has its own interface — the ZIP is always the same.

4
Configure

Set completion tracking rules

In your LMS's SCORM settings, specify how the system marks a course "complete." Options: Passed/Failed (based on a quiz score), Completed (based on slide progress), or simply Viewed (the learner launched it). Always test with a dummy learner account before releasing to your full audience.

5
Launch

Enroll learners and monitor progress

Enroll learners individually, by cohort, or via self-enrollment links. After launch, use the LMS Reports section (usually under Administration or Analytics) to see: completion rates, average quiz scores, time spent, and individual learner progress. Export as CSV and analyze in Google Sheets or Excel to share with stakeholders.

Best Practices

Accessibility in Instructional Design

Accessible design ensures every learner — regardless of disability — can fully engage with your content. Building accessibly from the start is faster and cheaper than retrofitting.

⚠️
It's often a legal requirement, not just best practice

In the US, Section 508 and the ADA require digital learning to be accessible in federal agencies and organizations receiving federal funding. In the EU, the European Accessibility Act applies broadly. The international standard is WCAG 2.1 Level AA. Designing to AA keeps you compliant and inclusive.

1
Visual

Check your color contrast

Text must achieve at least a 4.5:1 contrast ratio against its background (3:1 for large text over 18pt). Never use color as the only way to convey information — if you show correct answers in green, also add a checkmark icon or the word "Correct." Use WebAIM Contrast Checker (webaim.org/resources/contrastchecker) to verify every text/background pair in your course.

2
Visual

Write alt text for all images

In Storyline: right-click any image → Accessibility → Alt Text field. Write a description of what the image shows and why it matters in context. For decorative images (pure design elements): check "This object is decorative and can be ignored by screen readers." For complex charts or diagrams, provide a full text description below the image.

3
Audio / Video

Add closed captions to all audio and video

In Storyline: select your audio object → Insert → Closed Captions to auto-generate and then manually edit for accuracy. For video, start with auto-captions from YouTube, correct errors, then export the .srt file. Always provide a text transcript as an alternative — it also helps SEO and learners who prefer reading. Captions benefit all learners, not only those who are deaf or hard of hearing.

4
Navigation

Enable full keyboard navigation

Every interactive element must be reachable and activatable using only a keyboard: Tab to move between elements, Enter or Space to activate. In Storyline: Home → Tab Order — set this for every slide, listing all elements in the correct reading sequence. Test by closing your mouse completely and navigating your entire course with keyboard alone.

5
Content

Write accessible, plain-language content

Use active voice and plain language. Avoid jargon, or define it when unavoidable. Structure all content with a logical heading hierarchy: H1 (course title) → H2 (module) → H3 (lesson) → H4 (slide topic). Never describe content by its position alone ("see the chart on the right") — screen readers can't see position. Instead, describe what the content conveys: "The bar chart shows sales grew 40% in Q3."

6
Test

Test with a real screen reader

Download NVDA (free for Windows at nvaccess.org) or use VoiceOver (built into Mac: Cmd+F5, and iOS). Navigate your published course with the screen reader active and listen to how every element is announced. Fix elements that are skipped, mislabeled, or announced confusingly. Also run the published HTML through WAVE (wave.webaim.org) for an automated accessibility report.
